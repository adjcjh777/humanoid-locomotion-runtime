发现 play/Viser 中斜走现象

对 model_4500.pt 等 checkpoint 的回放观察显示在 Viser 中经常斜着走。这触发了后续策略调整：不能只按最终 reward 或单次回放观感选 checkpoint，需要固定 command-grid eval 来量化 lateral drift、yaw error、velocity error 和 done fraction。

2026-06-30：训练策略重构和多 seed 实验

代码侧新增能力

  • 新增 src/humanoid_locomotion_runtime/unitree_g1_23dof_profiles.py，在不修改 Unitree submodule tracked source 的情况下注册 repo-local MJLab tasks。
  • 新增 Unitree-G1-23Dof-ForwardFlat，用于 Stage A fixed-straight/stand 稳定性训练。
  • 新增 Unitree-G1-23Dof-VelocityBalancedFlat，用于 Stage B 通用 velocity controller。
  • 新增 binned velocity command sampler，明确采样 stand、straight_forward、yaw_only、lateral_only、combined。
  • 新增 command_axis_leakage_penalty，约束命令为 0 的轴上不要出现实际速度串扰。

工具侧新增能力

  • scripts/eval_unitree_g1_23dof_command_grid.py：固定 stand/forward/yaw/lateral command grid 评估 checkpoint。
  • scripts/summarize_unitree_g1_23dof_eval.py：汇总 eval JSON，支持 --group-by checkpoint 和 seed allowlist。
  • scripts/run_unitree_g1_23dof_stage_b_eval_queue.sh：训练结束后自动等待 final checkpoint，再评估关键 checkpoints。
  • scripts/run_unitree_g1_23dof_training.sh：默认 task 切到 VelocityBalancedFlat，SAVE_INTERVAL=250，支持显式 seed。
