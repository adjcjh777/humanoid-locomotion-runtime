## 1. Current diagnosis：必须守住的主线

1. **定位是 supervisor，不是 controller。** 策略只在 8 个高层动作里选择，绝不能滑向输出 joint/gait/action 的低层控制，否则直接撞上成熟运动控制与 HumanoidBench 的红海。
2. **唯一可辩护的 novelty 是「typed supervisory memory」。** 即在监督层上，结构化的 event-trace / body-state 记忆何时比 instant state 更有价值——这是对 HoRD、Chameleon 的正面切割线，不能泛化成通用 episodic memory。
3. **claim 必须是条件性的，而非绝对优势。** 论文核心问句是「在什么失败模式下 memory 带来增益」，不是「memory 永远更好」。这决定了 ablation 而非 SOTA 是主证据。
4. **baseline 阵列先于方法成立。** frozen-only、heuristic supervisor、VLM-prompt supervisor、no-memory 四条 baseline 是论文的地基，缺一条审稿人就能质疑。
5. **范围必须钉死在 MuJoCo + Unitree G1 单本体仿真。** 真机、多本体、open-vocab grounding 都是诱惑，会把 3 个月预算烧光。

## 2. Candidate ideas

**I1. Typed event-trace memory summary 作为监督输入（核心）**

将失败/状态信号压缩成有类型的事件序列（滑移、接触丢失、目标漂移…）喂给 supervisor。
novelty tension：与 HoRD/Chameleon 重叠，但限定在监督层 typed event 而非控制历史。可行性 5。可证伪点：no-memory 与 typed-memory 在长时序失败上的恢复成功率差。PRD 兼容 5。**评分 5**。

**I2. Body-state memory（本体趋势/退化记忆）**

记录关节温度漂移、跟踪误差累积等慢变量趋势。
novelty tension：中，趋势记忆较少人做在 supervisor。可行性 4。可证伪点：在「缓慢退化型」失败上 body-memory 是否优于 instant state。PRD 兼容 5。**评分 4**。

**I3. Memory-value diagnostic（何时记忆有价值的可解释指标）**

## 3. Recommended core package

选 **I1 + I3 + I10**，三者互锁形成论文骨架：

- **I10（失败注入协议）** 提供可复现、可分类的评测床，定义失败模式分布——没有它，所有 memory claim 都无法证伪。
- **I1（typed event/body memory）** 是方法主体，在监督层引入结构化记忆，直接对冲 HoRD/Chameleon。
- **I3（memory-value diagnostic）** 把贡献从「又一个 memory 方法」升级为「何时、为何记忆有价值」的分析性结论，这是审稿人记得住的句子。

论文叙事：在 I10 定义的失败谱上，I1 的 typed memory 仅在「长时序累积/缓慢退化」失败上系统性优于 instant-state supervisor，I3 量化并解释了这一条件边界。I2、I7、I8 作为支撑消融嵌入；I4、I5 作为对照基线。这样核心 claim 是条件性的、可证伪的、且不依赖打败成熟控制器。

## 4. Kill list（暂缓，不删除）

## 5. Experiment order and gates（4 周）

- **Week 1：搭 I10 + frozen-only baseline。** Gate：≥3 类可复现失败注入，frozen-only 在受控失败上有明确可测的失败率基线。No-Go：失败注入不可复现或全 0/全 100% → 重做协议。
- **Week 2：no-memory supervisor + heuristic supervisor。** Gate：学到的 no-memory supervisor 显著优于 heuristic（恢复成功率提升 ≥10 个百分点）。No-Go：学不过 heuristic → 检查动作空间/reward（回 I9/I5）。
- **Week 3：I1 typed memory + I7 horizon 扫描。** Gate：在至少 1 类长时序失败上，typed-memory 比 no-memory 提升 ≥10 点且统计显著。No-Go：全谱无差异 → claim 收窄或 pivot 到协议贡献。
- **Week 4：I3 diagnostic + I4 VLM-prompt baseline。** Gate：能定位出 memory 有价值的失败子集，且 learned supervisor 在延迟/成功率上对 VLM-prompt 有可辩护优势。No-Go：VLM-prompt 全面碾压 → 改写成「轻量 learned supervisor vs VLM」的效率论文。
